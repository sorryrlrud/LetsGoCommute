# 출근드림팀 / Let’s Go Commute

GPS 기반 이동 기록 서비스 **출근드림팀**의 1차 MVP입니다. 출발 후 이동 중 체크포인트를 찍고, 도착 뒤 체크포인트 이름과 구간별 이동수단을 입력해 내 이동 기록을 로컬에 저장하고 비교합니다.

## 1차 MVP 기능

- 출발, 체크, 일시정지, 재개, 도착 흐름
- 10초 간격 GPS 포인트 저장
- 일시정지 중 현재 위치 갱신, 이동 경로 저장 중단
- 도착 후 요약, 체크포인트/구간 편집, 저장 검증
- 기록 중/도착 후 체크포인트 확인, 편집, 삭제, 병합
- 집/회사 위치 유형과 체크포인트 장소 이력
- IndexedDB 로컬 저장과 내 기록 목록/상세/삭제
- 체크포인트 자동 저장 알림과 브라우저/PWA 알림 설정
- 출발지와 도착지가 각각 100m 이내인 기록 자동 추천 비교
- 구간 수가 다른 기록 비교와 `구간 구조가 다릅니다` 안내
- Leaflet + OpenStreetMap 지도
- manifest와 service worker를 포함한 PWA 기본 지원

## 사용 기술

- Vite
- React
- TypeScript
- Leaflet
- OpenStreetMap 타일
- IndexedDB
- PWA manifest / service worker

## 로컬 실행

```bash
npm install
npm run dev
```

## 빌드

```bash
npm run build
```

빌드 결과는 `dist/`에 생성됩니다. `vite.config.ts`의 `base: './'` 설정으로 GitHub Pages 하위 경로에서도 정적 파일 경로가 동작하도록 구성했습니다.

## GitHub Pages 배포

1. `npm run build`를 실행합니다.
2. `dist/` 디렉터리를 GitHub Pages 배포 대상으로 설정합니다.
3. GitHub Actions를 쓴다면 Node 설치 후 `npm ci`, `npm run build`, Pages artifact 업로드 순서로 배포하면 됩니다.

## 1차 MVP 제외 기능

회원가입, 로그인, Firebase, 서버 저장, 공개 리더보드, 다른 사용자 기록 탐색, 소셜 로그인, 닉네임 시스템, 관리자 기능, 정류장명/역명 자동 추천, 네이버/카카오/유료 지도 API, JSON 가져오기/내보내기, 체크포인트 순서 변경/위치 직접 수정, 서버 기반 원격 푸시, 백그라운드 GPS 기록 보장은 구현하지 않습니다.

## 개인정보 및 위치정보 주의

이 앱은 위치 기록을 브라우저 IndexedDB에 저장합니다. 서버로 전송하지 않지만, 같은 브라우저 프로필을 사용하는 사람은 기록을 볼 수 있습니다. 앱 삭제, 브라우저 데이터 삭제, 다른 기기 사용 시 기록이 사라지거나 공유되지 않을 수 있습니다.

## OpenStreetMap 타일 주의

1차 MVP 검증용으로 OpenStreetMap 공용 타일을 사용합니다. 대량 트래픽 서비스에는 별도 타일 서버나 적절한 지도 제공자 검토가 필요합니다.

## Known Limitations

- 1차 MVP는 로컬 저장만 지원합니다.
- 브라우저 또는 기기를 바꾸면 기록이 공유되지 않습니다.
- 앱을 삭제하거나 브라우저 데이터를 삭제하면 기록이 사라질 수 있습니다.
- 백그라운드 GPS 기록은 보장하지 않습니다.
- GPS 정확도는 기기와 환경에 따라 달라집니다.
- OpenStreetMap 공용 타일은 대량 트래픽 서비스용이 아닙니다.
- 자차와 택시는 이동수단으로 제공하지 않습니다.
- 공개 리더보드와 다른 사용자 기록 탐색은 2차 기능입니다.
